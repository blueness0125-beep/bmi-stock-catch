## 1. 프로젝트 개요

한국 주식(KOSPI/KOSDAQ) 급등 종목을 자동 감지하여 **8항목 15점 만점 스코어링**으로 등급을 매기고, 대시보드에서 시각화하는 시스템이다.

### 핵심 기능

| 기능 | 설명 |
|------|------|
| 종가베팅 V2 | 8항목 15점 스코어링 → A/B/C 등급 분류 |
| VCP 스캐너 | Volatility Contraction Pattern 감지 및 등급 부여 |
| 마켓 게이트 | KODEX 200 이동평균 기반 시장 국면 판단 |
| 포지션 사이징 | R 기반 손절가/목표가/수량 자동 계산 |
| Gemini AI 뉴스 분석 | 뉴스 테마 추출 및 재료 강도 점수화 |
| 텔레그램 알림 | 시그널 생성 후 자동 알림 전송 |
| 대시보드 | Next.js 기반 실시간 조회 (Vercel 배포) |

```
   → 뉴스 수집 + Gemini AI 분석
   → 8항목 스코어링
   → 포지션 사이징
   → data/jongga_v2_latest.json 저장

2. VCP 스캐너 실행 (vcp_scanner.py)
   → 거래대금 상위 50~100개 비동기 차트 수집
   → VCP 패턴 감지
   → data/vcp_signals.json 저장

3. 프론트엔드 (Vercel)
   → Server Component에서 data-server.ts로 JSON 직접 읽기
   → 대시보드 렌더링
   → market-gate route에서 Naver Finance 실시간 스크래핑
```

## 4. Python 엔진

### 4-1. 데이터 수집 (`collectors.py`)

| 함수 | 수집 대상 | 출처 |
|------|-----------|------|
| `get_top_gainers(market, config)` | KOSPI/KOSDAQ 급등 종목 시세 | Naver Finance 모바일 API |
| `get_chart_data(code, days)` | 60일 OHLCV 일봉 + MA5/10/20 | Naver Finance 일봉 테이블 |
| `get_stock_news(code, name, limit)` | 종목 관련 뉴스 3건 | Naver 뉴스 |
| `get_supply_data(code)` | 외국인/기관 5일 순매수 | Naver Finance 외인동향 |

**필터링 조건** (`SignalConfig`):

```
거래대금 ≥ 500억
당일 등락률: 5% ≤ pct ≤ 20%
주가: 1,000원 ≤ price ≤ 500,000원
제외: ETF, SPAC, 우선주, 스팩, 인버스, 레버리지
```

### 4-2. LLM 분석 (`llm_analyzer.py`)

- 모델: `gemini-2.5-flash` (환경변수 `GEMINI_MODEL`)
- API 키: 환경변수 `GOOGLE_API_KEY`
- 입력: 종목명 + 뉴스 제목/요약 3건
- 출력:

```json
{
  "score": 0~3,
  "reason": "LLM이 판단한 재료 요약",
  "themes": ["반도체 소부장", "메모리 반도체"]
}
```

### 4-3. 스코어링 (`scorer.py`)

**8항목 15점 만점 체계:**

| # | 항목 | 배점 | 조건 |
|---|------|------|------|
| 1 | 뉴스/재료 | 0~3 | LLM 점수 우선, 없으면 뉴스 존재 여부 |
| 2 | 거래대금 | 0~3 | 1조↑=3, 5000억↑=2, 1000억↑=1 |
| 3 | 차트패턴 | 0~3 | 이평선 정배열(+1) + 52주신고가 or 60일 돌파(+1) |
| 4 | 캔들형태 | 0~1 | 몸통비율 ≥ 60%, 윗꼬리 ≤ 30% |
| 5 | 기간조정 | 0~1 | 20일 변동성 압축 + 상단 돌파 |
| 6 | 수급 | 0~2 | 외인·기관 양방향 순매수(2), 단방향(1) |
| 7 | 조정폭 회복 | 0~1 | 최근 10일 고점 → 저점 → 50% 이상 회복 |
| 8 | 되돌림 지지 | 0~1 | 과거 저항선 돌파 후 눌림에서 지지 확인 |

**등급 결정 기준:**

```
필수 조건: 뉴스 ≥ 1점 AND 거래대금 ≥ 1점
A등급: 총점 ≥ 9점
B등급: 총점 ≥ 7점
C등급: 총점 < 7점 또는 필수 조건 미통과
```

**품질 점수 (100점 만점):**

| 항목 | 최대 |
|------|------|
| 수급 쌍매수 | 30점 |
| 총점 구간 | 25점 |
| 당일 등락률 (낮을수록 우수) | 20점 |
| 20일 모멘텀 (낮을수록 우수) | 15점 |
| 거래량 비율 (4~6배 구간 최고) | 10점 |

**3중 품질 게이트** (기본 설정):

```
Gate 1: 수급 점수 ≥ 2
Gate 2: 총점 ≥ 8
Gate 3: 품질 점수 ≥ 55
```

### 4-4. 포지션 사이징 (`position_sizer.py`)

R 기반 포지션 사이징:

```
R값 = 자본 × 0.5%  (기본 자본: 1,000만원 → R = 5만원)

등급별 R배수:
  A등급: 2R  → risk_amount = R × 2
  B등급: 1R  → risk_amount = R × 1
  C등급: 0R  → 매매 없음

position_size = risk_amount ÷ 손절비율(5%)
최대 포지션 = 자본 × 50% 로 상한 제한

손절가 = 진입가 × (1 - 5%)
목표가 = 진입가 × (1 + 15%)
수량 = int(position_size ÷ 진입가)
손익비 = 15% ÷ 5% = 3.0
```

### 4-5. VCP 스캐너 (`vcp_scanner.py`, `vcp_detector.py`)

**Volatility Contraction Pattern 감지:**

1. 거래대금 상위 50~100개 종목 비동기 수집 (Semaphore=5 동시 처리)
2. 60일 일봉 데이터로 프랙탈 스윙 고/저점 탐색
3. 3회 수축 패턴 (C1 → C2 → C3) 확인
4. EMA20/60 추세 조건 확인

**VCP 등급:**

| 등급 | R12 최소 | R23 최소 | 추세 조건 | 고점 하강 | 저점 상승 |
|------|---------|---------|----------|---------|---------|
| A | 1.20 | 1.15 | EMA20 > EMA60 + EMA60 상승 | 필수 | 필수 |
| B | 1.12 | 1.08 | 현재가 > EMA20 | — | — |
| C | 1.05 | 1.03 | 현재가 > EMA60 | — | — |
| D | 1.02 | 1.01 | 조건 없음 | — | — |

**출력 결과** (`vcp_signals.json`):

### 5-3. API Routes

모든 API는 Flask 없이 Next.js 서버리스 함수로 동작한다.

| 경로 | 동작 |
|------|------|

### 5-4. 서버/클라이언트 데이터 접근

**`data-server.ts` (서버 전용):**
- `fs.readFileSync`로 `public/data/` JSON 직접 읽기
- Vercel에서 HTTP 자기 호출 불필요 → 콜드 스타트 없음
- Server Component 및 API Route에서만 임포트

```
data-server.ts
├── getSignals()         → jongga_v2_latest.json
├── getAvailableDates()  → 날짜별 파일 목록
├── getHistoryByDate()   → jongga_v2_results_YYYYMMDD.json
├── getVCPSignals()      → vcp_signals.json
└── getCumulativeStats() → 전체 날짜 파일 집계
```

**`api.ts` (클라이언트):**
- 상대 경로 `/api/kr/*`로 Next.js API Route 호출
- 주요 타입: `Signal`, `ScoreDetail`, `Checklist`, `NewsItem`, `VCPData`, `MarketGateResponse`

### 5-6. SignalCard 구성

```
SignalCard
├── 헤더: 종목명 + 등급 배지 + [신고가] [부정뉴스] 배지
├── 총점 바: X/15 · 품질 N점
├── 8항목 점수 필 (2행 × 4열)
│   뉴스/거래량/차트/캔들 | 눌림목/되돌림/수급/압축
├── 체크리스트 뱃지 (활성 항목만 표시)
│   MA정렬 / 좋은캔들 / 수급양호 / 되돌림복귀 / 눌림지지 / 압축구간 / 돌파 / 윗꼬리
```
